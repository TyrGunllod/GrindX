    2. Em `pytest.ini`, atualizar a linha `addopts` para incluir flags de cobertura. Estado atual:
       ```
       addopts = -v --tb=short --strict-markers
       ```

    5. Rodar testes para verificar que coverage funciona: `cd apps/api-postgres && set PYTHONPATH=..\..\packages && python -m pytest tests/ -v --tb=short --cov=app --cov-report=term-missing --cov-fail-under=70`

    ATENÇÃO: Se os testes falharem porque coverage está abaixo de 70%, isso é esperado — a configuração está correta, mas a cobertura atual pode ser insuficiente. Neste caso, documentar o percentual atual no SUMMARY e seguir para o próximo task. O threshold de 70% é o target, não precisa ser atingido agora (será atingido após adicionar mais testes em fases futuras).

    NOTA: Se `--cov=app` não funcionar (coverage 0%), tentar `--cov=apps/api-postgres/app` — o pytest roda do diretório de trabalho `apps/api-postgres/`, então `app` deve resolver corretamente. Se ainda não funcionar, usar `--cov=app` com `--cov-config=pytest.ini` explícito.
