<objective>
Temporary passwords use cryptographically secure generation (secrets module, 16 alphanumeric chars) with 15-minute expiry stored in the database. File upload endpoints validate magic bytes using the filetype library, rejecting files whose actual content doesn't match the declared type.

Purpose: Prevent credential stuffing via non-expiring temp passwords and prevent malicious file uploads disguised with wrong extensions.
Output: Updated auth service with expiry logic, Alembic migration for expires_at field, updated upload endpoints with filetype validation, tests for both features.
</objective>

    2. Create a helper function `_validate_file_magic(file_bytes: bytes, allowed_mimes: set[str], file_label: str) -> None` that:
       - Calls `kind = filetype.guess(file_bytes[:261])`.
       - If `kind is None`: raise `HTTPException(status_code=400, detail=f"Não foi possível detectar o tipo do arquivo de {file_label}. Verifique se o arquivo não está corrompido.")`.
       - If `kind.mime not in allowed_mimes`: raise `HTTPException(status_code=400, detail=f"Tipo de arquivo de {file_label} não corresponde ao conteúdo. Esperado: {', '.join(allowed_mimes)}. Detectado: {kind.mime}")`.

    3. Modify `upload_font` endpoint:
       - After the file extension check, read file content: `content = await file.read()`.
       - Call `_validate_file_magic(content, {"font/sfnt", "font/woff", "font/woff2", "application/octet-stream"}, "fonte")`.
       - Use `content` for saving instead of `shutil.copyfileobj(file.file, buffer)` — write `buffer.write(content)`.
       - Add `await file.seek(0)` before size check, or read size from `len(content)`.

    4. Modify `upload_logo` endpoint:
       - After the content_type check, read file content: `content = await file.read()`.
       - Call `_validate_file_magic(content, {"image/jpeg", "image/png", "image/gif", "image/webp"}, "logo")`.
       - Use `content` for saving.
       - Note: SVG files (`image/svg+xml`) are excluded from magic bytes validation because filetype may not detect them reliably — keep the existing content_type check for SVG and add a comment: `# SVG não suportado por magic bytes — validação por content_type`.
