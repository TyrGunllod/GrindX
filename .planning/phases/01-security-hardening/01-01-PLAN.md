  <action>
    **api-postgres/app/core/config.py — per D-01 (SEC-01):**

    1. Add `import math` and `from collections import Counter` at the top.
    2. Replace the existing `validar_secret_key` field_validator with one that:
       - First checks `len(v) < 32` (keep existing message).
       - Then calculates Shannon entropy: `freq = Counter(v); length = len(v); entropy = -sum((count/length) * math.log2(count/length) for count in freq.values())`.
       - If `entropy < 3.5`, raises `ValueError` with message: `f"SECRET_KEY tem entropia muito baixa ({entropy:.2f} bits/caractere). Use uma chave aleatória: python -c \"import secrets; print(secrets.token_hex(32))\""`.
    3. Add `ENVIRONMENT: str = "development"` field after `DEBUG`.
    4. Add `@property def is_production(self) -> bool:` returning `self.ENVIRONMENT == "production"`.
    5. Replace the existing `allowed_origins_list` property — per D-05 (SEC-05):
       - If `self.is_production` and `not self.CORS_ORIGINS.strip()`: raise `ValueError("CORS_ORIGINS obrigatório em produção. Defina origins explícitos (ex: https://app.grindx.com)")`.
       - If `self.is_production` and origins contain `"*"`: raise `ValueError("CORS_ORIGINS não pode ser '*' em produção")`.
       - Parse origins from `self.CORS_ORIGINS` using the existing `_parse_origins` helper logic (clean brackets/quotes, split by comma, strip each).
       - In non-production mode with empty `CORS_ORIGINS`, return `["http://localhost:3000", "http://localhost:5500", "http://127.0.0.1:5500"]`.

    **api-sqlserver/app/core/config.py — per D-01 (SEC-01) and D-05 (SEC-05):**

    4. Apply the same Shannon entropy validator to `SECRET_KEY`.
    5. Change `CORS_ORIGINS: list[str] = ["*"]` to `CORS_ORIGINS: str = ""` (string, same as api-postgres).
    6. Add `ENVIRONMENT: str = "development"` field.
    7. Add `is_production` property (same as api-postgres).
    8. Replace `allowed_origins_list` property with the same production/dev logic as api-postgres (reuse the same parse-and-validate pattern).

       - `test_cors_dev_allows_empty`: Set ENVIRONMENT="development", CORS_ORIGINS="". Assert allowed_origins_list returns localhost defaults.
    10. Use `import secrets` for generating test keys. Use `monkeypatch` or direct Settings instantiation for env var overrides.
